## Type Safety Improvement

### BEFORE - Untyped String
```csharp
// Server sends: string Payload
// Could be anything!
string payload = JsonSerializer.Serialize(sessionObj);

// Client receives: string Payload
// No compile-time guarantee of format
var sessionData = JsonSerializer.Deserialize<SessionRecoveryData>(payload);
// Runtime crash if wrong type! ??
```

### AFTER - Typed DTO
```csharp
// Server explicitly creates typed object
var recovery = new SessionRecoveryData
{
    TimeRemainingSeconds = 3600,
    Balance = 350000
};
string payload = JsonSerializer.Serialize(recovery);

// Client explicitly deserializes typed object
var sessionData = JsonSerializer.Deserialize<SessionRecoveryData>(payload);
// Compile-time guarantee: correct type! ?
// Runtime deserialization verified! ?
```

## Message Structure

### Request (ASCII-based)
```
{
  "Action": "SessionRestore",
  "Payload": "{\"UserId\": 1, \"ComputerId\": 5, \"TimeRemainingSeconds\": 3600}"
}
```

### Response (ASCII-based)
```
Success:
{
  "Action": "SessionRestore",
  "Payload": "{\"TimeRemainingSeconds\": 3600, \"Balance\": 350000}"
}

Error:
{
  "Action": "SessionRestore",
  "Payload": "Error: User not found"
}
```

Both serialized to JSON line format (single line per message) sent over TCP.
